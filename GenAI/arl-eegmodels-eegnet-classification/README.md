# ARL EEGModels EEGNet Evaluation Support

This folder is not the main contribution of the repository. It contains ARL EEGModels/EEGNet-based evaluation code used after EEG samples are generated by the primary eeg-diffusion-generation/ project.

The purpose of this folder is to test generated EEG through downstream classification experiments. It keeps the ARL EEGModels implementation of EEGNet, DeepConvNet, and ShallowConvNet together with local notebooks and scripts for EEGNet training, k-fold validation, generated-data augmentation, subject-level evaluation, and result plotting.

## Repository Layout

- src/ - reusable model definitions, including EEGModels.py.
- scripts/ - Python experiment scripts for EEGNet training, generation-assisted augmentation, and k-fold evaluation.
- notebooks/ - exploratory and evaluation notebooks from the research workflow.
- results/ - exported plots and accuracy summary figures.
- data/ - placeholder for local datasets and result pickles that are not committed.

## Main Workflows

1. Train EEGNet variants as downstream classifiers.
2. Compare real-only training against generated-data augmentation and averaged generated-data settings.
3. Summarize subject-level classification accuracy using saved result pickles and plotting notebooks.
4. Use classification results as one evaluation signal for generated EEG quality.

## Dataset and Evaluation Preprocessing

This evaluation module uses the same Cho et al. 2017 motor imagery EEG dataset used by the generation project. The ARL EEGModels/EEGNet code is applied after preprocessing to test whether generated EEG helps or preserves useful class information for left-hand versus right-hand motor imagery.

Dataset reference: https://academic.oup.com/gigascience/article/6/7/gix034/3796323

Evaluation preprocessing in this folder includes:

1. Use preprocessed subject-level EEG arrays derived from the original dataset.
2. Form binary labels for left-hand and right-hand motor imagery.
3. Use band-processed EEG tensors for EEGNet evaluation. Some scripts evaluate 32-channel versions by selecting every other channel from the preprocessed tensors.
4. Scale EEG values for model input and format tensors as trials, channels, samples, kernels for EEGNet.
5. Run stratified train/validation/test splits and repeated k-fold style EEGNet evaluation.
6. Compare real-only training with generated-data augmentation or averaged generated-data settings.

The dataset, derived arrays, checkpoints, and result pickles are not included in this GitHub-ready copy. Without obtaining and preprocessing the original Cho et al. dataset, these notebooks and scripts should be read as documentation of the evaluation workflow rather than as immediately reproducible experiments.

## Setup

1. Create a Python virtual environment.
2. Activate the environment.
3. Install dependencies from requirements.txt.

For scripts that import EEGModels, run from this project root and add src/ to PYTHONPATH.
