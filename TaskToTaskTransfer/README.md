# Task to Task Trasnfer Learning

MATLAB code for FgMDM-based within-task and task-to-task transfer decoding across **motor execution (ME)**, **motor imagery (MI)**, and **motor observation (MO)** EEG tasks.

This repository accompanies the following study:

> Daeun Gwon and Minkyu Ahn,
> **Motor task-to-task transfer learning for motor imagery brain-computer interfaces**,
> *NeuroImage*, 302, 120906, 2024.
> DOI: `10.1016/j.neuroimage.2024.120906`

## Overview

Motor imagery brain-computer interfaces (MI-BCIs) typically require users to perform repeated motor imagery trials during a calibration phase. This calibration process is often tedious, time-consuming, and cognitively demanding.

This project investigates whether EEG data from easier or more intuitive motor tasks can be used to reduce the calibration burden for MI-BCI.

The main idea is:

```text
Train on easier motor tasks
        ↓
Motor execution (ME) or motor observation (MO)
        ↓
Decode motor imagery (MI)
```

The repository focuses on FgMDM-based classification analyses:

1. **Within-task decoding**

   * Train and test within the same task.
   * Used as the baseline performance.

2. **Task-to-task transfer decoding**

   * Train on one task and test on another task.
   * Used to test whether ME/MO data can support MI decoding.

## Study Design

The original experiment included three motor tasks:

| Abbreviation | Task              |
| ------------ | ----------------- |
| ME           | Motor execution   |
| MI           | Motor imagery     |
| MO           | Motor observation |

Each task includes two classes:

| Label | Class           |
| ----- | --------------- |
| 1     | Left-hand task  |
| 2     | Right-hand task |

Expected experimental structure:

```text
Participants: 28
Tasks: ME, MI, MO
Sessions per task: 2
Trials per task: 100
Trials per class: 50 left + 50 right
Sampling rate: 300 Hz
EEG system: DSI-24 dry EEG system
```

## Analysis Pipeline

The classification pipeline is:

```text
Input .mat file
        ↓
Load EEG, trigger, sampling rate
        ↓
Band-pass filtering: 5–40 Hz
        ↓
Epoching: -2 s to +5 s around cue onset
        ↓
Separate left/right trials
        ↓
Compute covariance matrices from sliding windows
        ↓
Trace normalization
        ↓
FgMDM classification
```

Sliding window settings:

```text
Epoch range: -2 s to +5 s
Window length: 3 s
Step size: 0.2 s
Number of windows: 21
```

Example time windows:

```text
Window 1:  -2.0 s to  1.0 s
Window 2:  -1.8 s to  1.2 s
...
Window 21:  2.0 s to  5.0 s
```

## Classification Analyses

### 1. Within-task decoding

Within-task decoding trains and tests the classifier within the same task.

Examples:

```text
ME train → ME test
MI train → MI test
MO train → MO test
```

Purpose:

```text
Estimate baseline left/right decoding performance for each task.
```

Expected output shape:

```text
accuracy(time_window, subject_or_session_index, summary_metric)
```

Typical summary metrics:

```text
1 = maximum accuracy
2 = mean accuracy
3 = minimum accuracy
4 = standard deviation
```

Note: for formal reporting, the mean accuracy across folds/repeats is recommended as the primary metric. Maximum accuracy should be treated as an exploratory or window-selection metric.

### 2. Task-to-task transfer decoding

Task-to-task transfer trains the classifier on one task and tests it on another task.

Transfer pairs:

```text
ME → MI
ME → MO
MI → ME
MI → MO
MO → ME
MO → MI
```

Purpose:

```text
Evaluate whether data from one motor task can support decoding of another motor task.
```

Expected output shape:

```text
accuracy(subject, train_time_window, test_time_window, transfer_pair)
```

Transfer-pair index:

```text
1 = ME → MI
2 = ME → MO
3 = MI → ME
4 = MI → MO
5 = MO → ME
6 = MO → MI
```

Each transfer result is a `21 × 21` time-generalization matrix:

```text
rows    = training time windows
columns = test time windows
```

## Data Format

Raw EEG data are not included in this repository.

Each input `.mat` file is expected to contain at least:

| Variable      | Description                                         |
| ------------- | --------------------------------------------------- |
| `prepro_data` | EEG data matrix, expected shape: channels × samples |
| `event`       | Trigger vector                                      |
| `srate`       | Sampling rate                                       |
| `chanlocs`    | Channel information, optional but recommended       |

Expected trigger labels:

| Trigger value | Meaning         |
| ------------- | --------------- |
| `1`           | Left-hand task  |
| `2`           | Right-hand task |

The scripts assume preprocessed EEG files compatible with the original MIMOME experiment structure.

## Configuration

Copy the example configuration file:

```matlab
copyfile('config_example.m', 'config_local.m');
```

Then edit paths in `config_local.m`:

```matlab
cfg.data_dir = 'path/to/your/data';
cfg.result_dir = 'path/to/results';

cfg.fieldtrip_dir = 'path/to/fieldtrip';
cfg.adargc_dir = 'path/to/AdaRGC-master';
cfg.covtoolbox_dir = 'path/to/covariancetoolbox';
```

Typical analysis parameters:

```matlab
cfg.bandpass = [5 40];

cfg.epoch_tmin_sec = -2;
cfg.epoch_tmax_sec = 5;

cfg.cov_window_sec = 3.0;
cfg.cov_step_sec = 0.2;
cfg.n_time_windows = 21;

cfg.n_repeats = 10;
cfg.n_folds = 10;

cfg.method = 'Supervised';  % or 'normal', depending on the FgMDM implementation
```

## How to Run

### 1. Check dependencies

```matlab
check_dependencies
```

### 2. Run within-task decoding

```matlab
run_within_fgmmdm
```

This script estimates left/right classification accuracy within each task:

```text
ME → ME
MI → MI
MO → MO
```

### 3. Run task-to-task transfer decoding

```matlab
run_cross_fgmmdm
```

This script computes task-to-task transfer accuracy:

```text
ME → MI
ME → MO
MI → ME
MI → MO
MO → ME
MO → MI
```

### 4. Summarize results

```matlab
summarize_results
```

This script generates summary files and figures in the `results/` folder.

## Important Notes

### Supervised FgMDM option

The `Supervised` option follows the external FgMDM/AdaRGC implementation.

Before interpreting results as strict zero-shot or zero-training transfer, users should verify whether the selected FgMDM option uses only training labels or performs any form of supervised adaptation involving target labels.

For conservative interpretation:

```text
- Use non-adaptive or label-free settings for strict zero-shot transfer.
- Treat supervised adaptation as label-informed or oracle-style analysis if target labels are used.
```

### Maximum accuracy

Some original scripts store the maximum accuracy across time windows. This is useful for reproducing the reported optimal-window analysis.

However, for robust reporting, users should also inspect:

```text
- mean accuracy across folds/repeats
- subject-level accuracy distributions
- time-resolved accuracy curves
- transfer-pair-specific performance
```
